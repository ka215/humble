Sloth

Mobile First CSS Micro Framework. It has reduced styling coding costs to a minimum with non-BEM and simpler naming conventions than OOCSS.

github

Why Sloth?

We're tired of complicating the HTML hierarchy to apply the preferred style and marking up several class names broken up into features. CSS should be simpler and better. This framework was developed for those who want to shirk neglect one's duties.

The sloth is lightweight as it saves the effort to rely on the legacy browser. Also, Sloth can use advanced styling technology that utilizes flexbox, grid, dialog, and limited vendor prefixes. It supports SCSS and postCSS, and you can customize your build with the included gulp file.

Get Started Sloth!

Marking up in the Sloth style is very easy. Basically, you can complete most of your work simply by declaring the .sloth class to the parent element you want to apply style to. 
<body class="sloth">

</body>
OR
<div class="sloth">

</div>
Also, if you do not want to apply a style to a specific element in a Sloth style block, specify the .nons class in the not sloth style declaration. 
<body class="sloth">
  <h1 class="nons">Not Sloth Styled headline</h1>
</body>
